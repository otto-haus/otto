The boundary

The agent owns the steps. The human owns the doors.

Like a protected branch: build and test all day, but the merge is yours. otto runs reversible work and stops only at what you can't take back.

Two-way doors

The agent acts.

Reversible work runs without interruption. The reversible surface widens as the agent earns trust.

One-way doors

The human ratifies.

Irreversible work stops for a person. That door never relaxes.

A value that cannot refuse you is decoration.

02Set it up yourself
brew install go-task
git clone https://github.com/otto-haus/otto.git
cd otto
bun install
bun run install-extension   # then run /reload in Letta Code

otto runs as a behavior layer over Letta in local mode. The only hard dependency is Letta Code: no Docker, no database, no server. Humans need Bun and go-task for the task shortcuts.

Thesis

Culture is a test suite.

AI can already do the work, and do it alone. The only question is whether it will do it the way your company would: your standard, your definition of done, the corner you'd never cut. And if not, why not.

That is not a capability problem. It is a conduct problem.

otto is CI for agent behavior. In software, a bug fix becomes a test that blocks the bug forever. otto does the same for conduct: you correct an agent once, you ratify the rule, and from then on "done" is gated. Proof mapped to the work, or it isn't done. The mistake isn't discouraged. It is blocked.

correction→proposed rule→you ratify→executable check→receipt

Memory is what an agent knows. Culture is what it does under pressure.

You only get to say that once you've watched it run.
