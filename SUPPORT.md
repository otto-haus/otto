# Support

otto is an early, local-first project (v0.1). Support is community-driven and best-effort —
please be patient and kind. Here's where to take different needs:

## Questions, ideas, and discussion

Join the [Discord](https://discord.gg/hab9ZvbPH). It's the fastest place for usage questions,
ideas, and field notes.

## Bugs and feature requests

[Open a GitHub issue](https://github.com/otto-haus/otto/issues). Please search existing issues
first to avoid duplicates. Ground reports in what you actually observed — for bugs, include
your version/commit, steps to reproduce, and what you expected (receipts beat assertions).

## Security vulnerabilities

**Do not** open a public issue for security problems. Follow [`SECURITY.md`](SECURITY.md) —
use GitHub private vulnerability reporting or contact the maintainer through the project's
private channels.

## Finding your way around the docs

- [`README.md`](README.md) — what otto is, the core concepts, and current status
- [`docs/`](docs/) — architecture, install, runtime, autonomy, practices, standards, and more
- [`CONTRIBUTING.md`](CONTRIBUTING.md) — how to propose changes
