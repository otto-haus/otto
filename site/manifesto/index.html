Manifesto

otto

The behavior OS for agentic work.

Agents can do the work. They still repeat corrected mistakes.

otto exists to make corrections stick.

correction → ratified check → future run → receipt

Paperclip is the office: goals, tickets, workers, budgets, heartbeats.

Letta is memory: identity, context, lessons.

Files are the record: durable truth.

otto is the operator workspace and behavior kernel: what can run, what needs a human, what counts as done, and what becomes future behavior.

AI capability is abundant. Trusted execution is scarce. A company of agents does not need more prompts. It needs an operating layer: permissions, standards, reviews, receipts, and memory that changes behavior.

otto turns lived corrections into executable culture. If an agent says “done” without proof, the correction becomes a check. Future done claims must attach evidence mapped to the work, or they fail. The mistake is not discouraged. It is blocked.

The doctrine is simple:

Agents own reversible work.
Humans own irreversible doors.
Done requires proof.
Every run leaves a receipt.
Every repeated correction can become a rule.

otto is not another agent shell, memory store, or task manager.

Letta remembers.
Paperclip manages.
otto improves.

The goal is not to ask the human more questions. The goal is to ask only the right ones: merge, publish, spend, send, change canon, touch live data.

The north star:

accepted useful work per hour of human attention

The first proof:

one rejection changes the next run

Culture is a test suite. otto is where it runs.

otto makes agent behavior compound. MIT · local-first · Letta runtime.
