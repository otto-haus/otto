The boundary

The agent owns the steps. The human owns the doors.

Like a protected branch: build and test all day, but the merge is yours. otto runs reversible work and stops only at what you cannot take back.

Two-way doors

The agent acts.

Reversible work runs without interruption. The reversible surface widens as the agent earns trust.

One-way doors

The human ratifies.

Irreversible work stops for a person. That door never relaxes.

A value that cannot refuse you is decoration.

Paste this into Claude Code, Codex, Cursor, or any coding agent. It clones otto, wires the Letta Code extension, and verifies the install.

# paste into your coding agent

02Set it up yourself
brew install go-task
git clone https://github.com/otto-haus/otto.git
cd otto
bun install
bun run install-extension   # then run /reload in Letta Code

Download otto and it works. The desktop app bundles and supervises its own Letta runtime, so the default “This Mac” path needs no separate Letta install. Prefer your own runtime? Point otto at an existing local Letta in advanced setup. Either way, provider/API keys live in Letta — never in otto.

Thesis

Culture is a test suite.

Most teams write their agent culture down — a contributing guide, a style doc, a list of “please always.” The agent reads it, nods, and forgets it by the next task. A value that lives only in a document is a value the agent can skip.

otto makes the rule executable. A correction becomes a check. The check runs on the next task, and the one after that. The lesson stops depending on whether anyone remembered to repeat it.

The things you care about are no longer posted on a wall. They are enforced on every run, with a receipt that shows what changed and why. A standard you can’t watch fail is just a hope.

Without otto

The correction evaporates.

correction ↓ forgotten ↓ repeated next task
With otto

The correction compounds.

correction ↓ ratified check ↓ enforced every run · receipt

You only get to say that once you’ve watched it run.
