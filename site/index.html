CI for agent behavior

Your agents can do the work. They still repeat corrected mistakes.

otto turns corrections into ratified checks, so future work must pass before the agent can call it done.

What it looks like

Teach it once. Make it hold.

Your agent claims a task is done without proof. You correct it. “Stop calling it done without the test output.”

Step 01

A correction arrives

It enters Curation before the behavior compounds.

Step 02

otto proposes a check

“Done” now requires proof.

Step 03

You ratify it

The check becomes a Standard the agent runs against itself. A suggestion is never canon.

The boundary

The agent owns the steps. The human owns the doors.

Like a protected branch: build and test all day, but the merge is yours. otto runs reversible work and stops only at what you can't take back.

Two-way doors

The agent acts.

Reversible work runs without interruption. The reversible surface widens as the agent earns trust.

One-way doors

The human ratifies.

Irreversible work stops for a person. That door never relaxes.

A value that cannot refuse you is decoration.

Install

Built to be installed by an agent.

01Have your agent install it

Paste this into Claude Code, Codex, Cursor, or any coding agent. It wires otto over your Letta runtime and verifies the install.

# paste into your coding agent
Retrieve and follow the instructions at:
https://raw.githubusercontent.com/otto-haus/otto/main/INSTALL_FOR_AGENTS.md
02Set it up yourself
brew install go-task
git clone https://github.com/otto-haus/otto.git
cd otto
bun install
bun run install-extension   # then run /reload in Letta Code

otto runs as a behavior layer over Letta in local mode. The only hard dependency is Letta Code: no Docker, no database, no server. Humans need Bun and go-task for the task shortcuts.
