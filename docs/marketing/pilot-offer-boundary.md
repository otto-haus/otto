# Pilot offer — claim boundary

## Category sentence

otto is the **behavior/culture layer** over Letta memory — standards, approvals, receipts, ratification — not a prettier Letta client or task manager.

## In scope (Managed Private Pilot)

- Install and operate one persistent agent workspace
- Receipts, approvals, memory writeback governance
- Weekly behavior improvement loop
- Optional hosted runner setup when scoped (086)

## Out of scope

- Multi-user SaaS, 24/7 SLA, guaranteed task completion
- Provider cost inclusion unless written SOW
- otto storing provider API keys (auth lives in Letta/keychain)
- Full Paperclip replacement

## Forbidden claims (pricing + sales)

Do **not** say or imply:

- Autonomous outcomes without human ratification
- otto replaces Letta or hosts provider keys
- “Done” without proof / receipts
- otto approves, verifies, guarantees, or makes work safe to ship
- Self-serve SaaS signup, free trial workspace, uptime SLA

## Letta billing clarity

Customer Letta/provider accounts and model usage are billed separately from the otto pilot fee unless explicitly scoped in writing.

## Human ratification

Pilot improves the behavior loop; the customer (Sebastian / office) retains Done decisions.
