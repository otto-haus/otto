## Summary

Closes the ratification → Letta write path for two north-star legs:

- **#639 (memory writeback):** A ratified `memory` proposal now PATCHes the targeted Letta
  core-memory block via `MemoryStore.updateBlock()`. Previously `decide(accept)` marked the
  proposal `applied` with no Letta call at all (`requiresCanonApply()` excluded `memory`).

- **#637 (behavior changelog injection):** Every ratification now injects a behavior-changelog
  snapshot into a dedicated `otto_behavior_changelog` Letta core-memory block so the agent's
  *next turn* sees the ratified change. Previously the changelog was UI-only; `letta-runner.ts`
  had zero changelog references.

### Architecture

- New `RatificationApplier` (`ratification-apply.ts`) owns side-effecting Letta writes.
  `ProposalStore.decide()` stays **pure file-state** (existing tests unchanged).
- Wired into `otto:curation:proposals:decide` IPC handler, **Labs-gated**
  (`memory_observatory`).
- Honest receipts: real Letta write → `success`; unreachable runtime or missing block label →
  `blocked` with recoverable `next_action`.
- `RatificationApplier` accepts a `MemoryWriter` interface — tests inject a hermetic fake so
  no live Letta agent is required.
