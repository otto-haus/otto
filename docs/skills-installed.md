# Installed agent skills (otto harness)

Project-level harness skills live under `.agents/skills/`. They extend Cursor/Codex agents working in this repo.

Letta charter skills (discord, github, routine, etc.) remain under `skill/` and are loaded by the Letta runtime — not covered here.

Install lockfile: `skills-lock.json` (generated by `npx skills add`).

## Install method

```sh
cd /path/to/otto
npx skills add <owner/repo> --skill <name> -y --copy
```

- `--copy` copies skill files into `.agents/skills/<name>/` (committed to git; no symlinks).
- Reinstall or update: `npx skills update` or re-run `npx skills add` with the same source.
- Discover more: load the `find-skills` skill or run `npx skills find <query>`.

## How agents invoke skills

Cursor and Codex auto-load skills from `.agents/skills/` when descriptions match the task. Explicit invocation:

```txt
Read .agents/skills/<name>/SKILL.md and follow it.
```

Examples:

```txt
Read .agents/skills/find-skills/SKILL.md — find a skill for PDF form filling.
Read .agents/skills/remotion-render/SKILL.md — render a 15s otto demo from TSX.
Read .agents/skills/just-scrape/SKILL.md — scrape this URL to markdown (check Bright Data first).
```

## find-skills — discover more

When you need capability you do not have installed:

1. Ask the agent to use `find-skills`, or run `npx skills find "<topic>"`.
2. Review the skills.sh page for source owner and Snyk/socket assessment.
3. Run skill-vetter before adding: `python3 ~/Code/omnara/codex/skills/skill-vetter/scripts/vet_skill.py .agents/skills/<candidate>`.
4. Install with `npx skills add <owner/repo> --skill <name> -y --copy` and update this doc + `skills-lock.json`.

## twitter-automation — claim boundaries

This skill documents inference.sh `belt` apps for X/Twitter. Otto gates apply:

- **Red (approval required):** post tweet, post with media, retweet, like, DM, follow, delete post.
- **Green (local/draft):** read tweet/user, draft copy locally, generate images/video for later review.
- **Spend:** `belt` / inference.sh usage consumes credits — treat as spend gate.
- **Secrets:** X credentials live in inference.sh account, not otto config.
- Do not imply otto posted on behalf of Sebastian without a receipt and explicit approval.

See `.agents/skills/twitter-automation/OTTO-GATES.md`.

## just-scrape — web scraping routing

When Bright Data MCP is connected (Cursor plugin), prefer Bright Data tools for search and scrape per workspace routing:

- Search → `search_engine` / `search_engine_batch`
- URL → markdown → `scrape_as_markdown`
- JS-heavy / interactive → `scraping_browser_*`

Use `just-scrape` when ScrapeGraph is explicitly requested, structured extract/crawl/monitor is needed, or Bright Data is unavailable. Always check credits (`just-scrape credits`) before bulk crawl. Respect rate limits and site ToS.

See `.agents/skills/just-scrape/OTTO-ROUTING.md`.
