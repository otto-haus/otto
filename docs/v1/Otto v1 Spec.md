# Otto v1 Spec

Otto is to culture what Letta is to memory.

Letta gives a persistent agent memory. Otto gives a persistent agent culture. It turns Standards, Practices, Routines, Approvals, Receipts, and corrections into governed future behavior.

Culture is not a page of values. Culture is what the agent repeatedly does under pressure. Otto makes that behavior explicit, ratified, repeatable, and improvable.

## Naming

- Product: **Otto**
- Public namespace: **otto-hq**
- Package scope: **@otto-hq/**
- Avoid exact public namespace: **Otto OS**

## Core doctrines

- A lesson is not culture until it changes future behavior.
- Long-term memory is only valuable if it changes future behavior.
- Own reversible work. Gate consequential work.
- Move fast on good judgment.
- Approve doors, not steps.
- Files are truth. UI is workspace. Memory is lessons.
- No artifact, no progress.
- Done requires mapped proof.
- External advice is input, not authority.
- Culture is what happens under pressure.
- One identity, many functions.

## Authority model

```txt
Sebastian -> Core Principles -> Standards -> Curation -> Practices/Routines/Charters/Channels/Memory
```

Sebastian owns consequences. Otto owns orchestration. Workers own bounded execution.

## v1 surfaces

### Standards
Explicit canon: what we reward, refuse, and do under pressure. Includes Core Principles and Precedents.

### Curation
Shared proposal-and-ratification engine. Curation decides what compounds across memory writebacks, Charters, Practices, Routines, Channels, Standards, Skills, and external actions.

### Approvals
Not a peer subsystem: Approval is the human-ratification record Curation emits when a proposal is consequential.

### Autonomy
Defines what Otto may own without Sebastian in the loop: ticket orchestration, worker management, worktrees, retries, checks, and safe integration steps.

### Charter
Operating contracts for autonomous work. Charters define the bet.

### Tickets
Bounded worker slices. Tickets define the slice; workers execute it; receipts prove it.

### Practices
Repeated behaviors worth preserving. Practices make Standards executable.

### Routines
Repeated bundles of Practices. Recurring routines require approval because attention is a one-way door.

### Channels
Communication surfaces. Discord is v0/v1 backend: mobile and ambient console, not source of truth.

### Skills
Reusable capability/context packages: 1Password, GitHub, Discord, PDFs, browser, Gmail/Calendar, etc.

### Knowledge
Maintained world model. v1 focuses on AI Frontier / Model Intelligence so Autonomy tracks actual capability.

### Desktop
Workspace over all of the above. It reads files as truth and shows status, queues, receipts, and decisions.

## v1 acceptance

Otto v1 is real when:

1. One canonical file contract exists for each surface.
2. Curation owns proposal/approval policy centrally.
3. Approvals are records emitted by Curation.
4. Desktop can show active Charters, Practices, Routines, Curation queue, Approvals, Runs, Receipts, Knowledge, and worker status.
5. Main Otto can orchestrate ticket workers in worktrees.
6. Receipts prove work and feed Curation/memory writebacks.
7. Standards appear in real Reviews and can block fake done.
8. Knowledge informs model routing and Autonomy.

## Final principle

Letta makes Otto remember. Otto makes memory change behavior.
