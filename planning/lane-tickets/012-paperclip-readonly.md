# 012 — Paperclip Readonly

Status: todo
Owner: future
Priority: P2
Depends on: 003, 016

## Outcome

Otto can read Paperclip work state without surrendering behavior authority.

## Scope

- Paperclip adapter, read-only.
- Import goals, issues, runs, worker outputs, blockers, and traces.
- Convert imported work into Otto receipts/context.
- Surface imported work in Runs/Receipts.

## Non-goals

- Otto does not create Paperclip tasks yet.
- Paperclip does not update Standards/Practices directly.
- Paperclip does not decide approvals.

## Done when

- Otto imports one Paperclip run.
- Imported work appears as a receipt/context packet.
- Source Paperclip IDs are preserved.
- Any behavior/canon implication becomes a Curation proposal.
- No Otto canon changes happen automatically.
